## Бот для приёма референсов и генерации через Kie.ai

Этот проект — Telegram-бот, который:
- **коммуницирует с сотрудником компании**;
- **принимает файлы/фото** в Telegram;
- **сохраняет их на сервере** в папку `bot/uploads/...`;
- готовит задачу и через **единый API Kie** (Kie.ai) отправляет данные на обработку.

### Единый API Kie: две модели

Всё идёт через **один API Kie** (один ключ, один базовый URL):

| Назначение | Модель в Kie | Что делает |
|------------|--------------|------------|
| **Промты и комментарии** | **GPT 5.2** | Обрабатывает замечания пользователя, дорабатывает/уточняет текст промта под задачу (режим process/similar). |
| **Генерация картинки** | **Nano Banana Pro** | По финальному промту и референсному фото генерирует изображение. |

То есть: базовый промт + комментарий пользователя → Kie (GPT 5.2) → уточнённый промт → Kie (Nano Banana Pro) + референсное фото → итоговое изображение.

### Стек

- Python 3.10+;
- `aiogram` 3.x — Telegram-бот;
- `httpx` — HTTP-клиент для API Kie;
- `python-dotenv` — загрузка переменных окружения из `.env`.

### Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте файл `.env` в корне проекта:

```bash
TELEGRAM_BOT_TOKEN=ваш_телеграм_токен

# Единый API Kie (и GPT 5.2, и Nano Banana Pro)
KIE_API_KEY=ваш_kie_api_key
KIE_BASE_URL=https://api.kie.ai

# Модели в Kie
KIE_LLM_MODEL=gpt-5-2
KIE_BANANA_MODEL=nano-banana-pro

# Чтобы результат генерации приходил пользователю в Telegram, поднимите callback-сервер и укажите его URL:
# KIE_CALLBACK_BASE_URL=https://your-domain.com
# CALLBACK_PORT=8765
```

Если `KIE_API_KEY` не указан, бот сохраняет файлы на диск, но вызовы Kie не выполняются.

### Запуск

**Бот (long polling):**

```bash
python bot/bot_main.py
# или: python -m bot.bot_main
```

**Callback-сервер** (чтобы Kie присылал готовое изображение в чат пользователю):

```bash
python -m bot.callback_server
```

Сервер слушает порт 8765 (или `CALLBACK_PORT`). Укажите в Kie публичный URL этого сервера в `KIE_CALLBACK_BASE_URL` (например `https://your-domain.com`), тогда бот будет передавать `callback_url` в createTask.

### Логика работы (режим Монолит / фундамент)

- Команда `/start`:
  - отправляет приветственное сообщение;
  - подсказывает использовать `/monolith`.

- Команда `/monolith`:
  - просит отправить **одно фото** монолитных работ/фундамента;
  - после получения файла спрашивает, что делать:
    - «Обработать текущее фото» (улучшить качество, свет, кадрирование);
    - «Новое, но похожее» (сгенерировать новый, но похожий кадр).

- Полный цикл (при настроенном `KIE_API_KEY`):
  - **Первый проход:** базовый промт из `prompt_builder` (без комментария) → сразу в **Nano Banana Pro** с референсным фото.
  - **После «Описать замечания»:** комментарий пользователя → **GPT 5.2** (промт-инженер) дорабатывает базовый промт → обновлённый промт → **Nano Banana Pro** с тем же фото.
  - Задача уходит в Kie (createTask), результат по callback (если поднят callback-сервер и указан `KIE_CALLBACK_BASE_URL`) приходит пользователю фото в чат.
  - Кнопки: «Подходит», «Описать замечания», «Перегенерировать».

Файлы в папке `bot/`:
- `bot/bot_main.py` — точка входа, регистрация клиентов Kie;
- `bot/monolith_flow.py` — сценарий монолит/фундамент, полный цикл;
- `bot/prompt_builder.py` — базовый универсальный промт;
- `bot/prompt_engineer.py` — доработка промта по комментарию через GPT 5.2;
- `bot/kie_llm_client.py` — клиент Kie Chat Completions (GPT 5.2);
- `bot/kie_banana_client.py` — клиент Kie createTask (Nano Banana Pro);
- `bot/job_store.py` — хранение job_id → chat_id для callback;
- `bot/callback_server.py` — HTTP-сервер для приёма результата от Kie и отправки фото в Telegram.
